Background

Here is one of the major projects I created for a “User Interaction Software” class. The purpose of the project was to use JavaScript, JQuery, and TypeScript in combination with HTML and CSS (of course) to create a web tool that would aid in research. More specifically, this tool would help researchers dynamically obtain and make sense of valuable data from a week’s worth of Tweets uploaded from RunKeeper, a popular running app. Simply put I was given a set of a few thousand Tweets uploaded from RunKeeper and had to make a website that would calculate and display certain statistics about the set of Tweets.

This project was a solo effort so all code contained in the GitHub repo was written by me. Furthermore, if you would like to play around with the project for yourself I have uploaded it here.

Design Choices

Since I knew that most of my allotted time on this project was going to be spent on the actual development of the site rather than its design, I aimed to create a simple but effective interface in order to be able to finish the project on time. I believe I accomplished this through displaying information about the set of Tweets in a coherent manner across each of the site's three pages.

Potential Improvements

I know this project was intended to just help us learn some front end development skills and not be used in an actual research project. However, I feel like with just a few additions and tweaks this project could actually be an effective research tool used in the real world.


1. Allowing users to upload their own sets of #RunKeeper Tweets

This is the key feature in turning this project into a research tool ready for real world use. Allowing users to upload their own sets of RunKeeper Tweets would open numerous doors of possibilities. Users would be able to upload their sets of Tweets and be able to gather data through the website in real time. Researchers could then easily compare different sets of Tweets against each other to gather valuable information (maybe people on the west coast bike more than people in other parts of the country, or people tend to run more in the morning than at night, etc). It all just depends on the type of RunKeeper Tweets the researchers want to learn about. Also, I feel like implementing this feature is very feasible since everything on the site is already dynamically generated according to the set of Tweets I was given by the professor. The set I was given is just every English Tweet with the Hashtag “#RunKeeper” in it uploaded to Twitter between Sunday, September 23, 2018 and Saturday, September 29, 2018.


2. Loading live Tweets

Another feature I feel that could be useful would be the ability to load RunKeeper Tweets in real time. This would update the set of Tweets used on the site with the most recent RunKeeper Tweets and update the site in real time as more RunKeeper Tweets are sent out. This would allow researchers to analyze the most recent RunKeeper trends as they happen. Implementing this feature could be accomplished through the use of a Twitter API.

What I Learned

Admittedly, this project was quite the grind and took me about 20 hours in total to complete since it was my first time using TypeScript and JQuery to dynamically generate content on a webpage according to a given dataset. However, grinding out this project was very rewarding in the end and gave me much more confidence in my overall programming abilities moving forward.

Through completing this project I gained the skill of using JQuery to dynamically modify web content with computed values (for example, calculating the number of tweets of each type of activity in the dataset). Furthermore, it enabled me to polish my HTML, CSS, and Bootstrap skills in general.
